## 3. Надиктовка упражнений — текущая архитектура (LIVE, #94+#95)
Конвейер наполнения библиотеки (НЕ-PII). Файлы: `backend/utils/exerciseStructuring.js` (все промпты + нормализация), `backend/services/structuringLlm.js` (LLM-клиент, provider-agnostic), `backend/routes/exercises.js` (`/structure`, `/plan-script`, `/transcribe`), `frontend/.../ExerciseModal.js` (UI), `frontend/src/hooks/useAudioRecorder.js`.
- **Планировщик скрипта** (`/plan-script`, `deepseek-v4-pro`): название/черновик/надиктовка → полный скрипт по чек-листу + `review_points`. Guard «Понял как: …» + флаг неоднозначности «X, а не Y» + дозировки арифметически согласованы + script чистый (уточнения только в review_points) + опирается на текст поля надиктовки.
- **Структуризатор** (`/structure`, `deepseek-v4-flash`): faithful, форма «вы», глоссарий упрощения в description, числа словами→цифры, guard на мусор.
- **3 advisory-критика ПАРАЛЛЕЛЬНО** (`Promise.allSettled` в `structureExercise`, review=true): клин.sanity + №5 полнота + №6 стиль/обращение. Все возвращают `{concerns:[{severity,field,message}]}` → один `summarizeSanity`. №6 **синонимы НЕ флагает** (квадрицепс/четырёхглавая/передняя поверхность — живой язык).
- **Ответы оператора**: к пунктам полноты/review_points — поле ответа + «Вписать ответы в текст надиктовки» → его слова в текст → пересборка (AI вставляет ответ, не выдумывает).
- **Инвариант:** источник клиники = эксперт. faithfulness-ревьюер видит raw_text → автофикс×1 → re-review.
