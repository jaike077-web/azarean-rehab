# PatientView — аудит и стабилизация UI

## Текущая структура
- Страница пациента: `frontend/src/pages/PatientView.js`.
- Стили страницы: `frontend/src/pages/PatientView.css`.
- Блок упражнения включает заголовок (бейдж + название + действия), видео-контейнер и параметры.

## Обнаруженные проблемы
- **Пересечения в заголовке упражнения:** бейдж и длинное название могли конфликтовать при переносах.
- **Нестабильный контейнер видео:** риски обрезки кадра на широких экранах.
- **NaN в параметрах:** некорректные числовые значения могли отображаться как `NaN`.

## Принятые решения
- Заголовок упражнения переведен на **CSS Grid** с колонками `44px / flex / auto`, добавлены правила для безопасного переноса текста и выравнивания действий.
- Для мобильных (`<= 640px`) заголовок складывается в две строки: первая — бейдж + название, вторая — блок действий на всю ширину.
- Контейнер видео стабилизирован: `aspect-ratio: 16/9`, `position: relative`, `overflow: hidden`, а вложенные элементы фиксированы через `position: absolute; inset: 0`.
- Добавлен helper `toInt(value, fallback)` для безопасного рендера чисел (sets/reps/duration/rest) и исключения `NaN`.

## Чек-лист ручной проверки
- [ ] Длинные названия упражнений не заходят под бейдж и не перекрывают кнопку.
- [ ] На мобильных заголовок упражнения делится на две строки, «Выполнено» не смещается.
- [ ] Видео отображается полностью (без обрезки краев) на широких экранах.
- [ ] В параметрах не появляется `NaN`; вместо него отображается корректный fallback.
