## Принципы исполнения

1. **Один файл = один коммит.** Не миксовать. Не открывать следующий файл, пока предыдущий не закрыт полностью (commit + smoke + push с одобрения Vadim'а).

2. **После каждого коммита обязательно:**
   - Все тесты зелёные локально (backend + frontend)
   - Smoke в реальном браузере (правило `feedback_smoke_real_browser.md`) — особенно для коммитов 02, 04, 05, 06, которые трогают UI
   - Push в `main` **только по явному «ок» от Vadim'а** (правило `feedback_no_direct_main_push_for_ui.md`)

3. **Между коммитами ⛔ STOP.** Это означает: после `git commit` финализировать запись в `wave_0_progress.md` (см. ниже), запросить smoke от Vadim'а, **дождаться** подтверждения, и **только потом** открывать следующий ТЗ-файл.

4. **Если коммит застрял** (тест падает / smoke ломает прод / нашлось что-то непредвиденное) — **остальные ждут**. Не «забегать вперёд».

5. **Никаких массовых push'ей подряд.** Прецедент `c8834b5` (11 push'ей за день, broken prod) — табу. Если возникает соблазн «давай сразу 2-3 коммита раз они мелкие» — это сигнал остановиться.
